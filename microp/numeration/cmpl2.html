Entiers signés

1.4 Code complément à 2

Principe

Les nombres positifs sont codés comme d'habitude. Les nombres négatifs sont représentés par : 

X < 0   -->   code(x) = 2n - |x|
En fait, le code complément à 2 est obtenu en incrémentant le code complément à 1 : 
Code2(x<0) = Code1(x<0) + 1
On déduit des propriétés du code complément à 1, que le code d'un négatif est simplement obtenu en incrémentant le complémentaire de sa valeur absolue : 
            _________    
Code(x<0) = Code(|x|) + 1
Exemple

La droite de 4 bits montre les codes des nombres -8 à +7 : 

    -8                      -1     0    +1                      +7   
-----+----- - - - - - - -----+-----+-----+----- - - - - - - -----+-->
   1000                    1111  0000  0001                    0111
Cette représentation incrémente le code des négatifs, et par là-même décale d'une unité vers la droite la demi-droite des négatifs par rapport au code complément à 1. Le problème de la double représentation de 0 est ainsi résolu de manière évidente. Il apparait également un négatif supplémentaire à la gauche de l'échelle. Par suite, 4 bits couvrent l'intervalle de -8 à +7, 8 bits couvrent [-128,+127] et 16 bits [-32768,+32767]. Plus générallement, n bits permettent de coder de -2n-1 à +2n-1-1.
 
Addition

L'addition de deux nombres positifs est classiquement effectuée en additionnant leurs codes.
 
Soient deux nombres de signe différent tels que : 
x > 0,  y < 0   /   |x| > |y|

En essayant de simplement additionner leurs codes : 
Code(x) + Code(y) = x + 2n - |y|                   
                  = 2n + (x+y)         avec x+y > 0
                  = 2n + Code(x+y)
ce qui conduit finalement à : 
Code(x+y) = Code(x) + Code(y) - 2n
Le dernier terme de cette équation ne modifie que le (n+1)ième bit du résultat, ce qui permet d'écrire : 
Code(x+y) = Code(x) + Code(y)
On laissera le lecteur examiner les deux autres combinaisons de signe pour x et y. Toutefois, la conclusion est que quels que soient les signes et les valeurs absolues respectives des deux nombres concernés, le code complément à 2 de la somme de deux nombres et obtenu en additionnant les codes complément à deux des deux opérandes.
 
La soustraction s'effectue par addition de l'opposé, et l'opposé est obtenu par : 
            ______     
Code(-x) =  Code(x) + 1
Dépassement de capacité

Il a été vu qu'en arithmétique signée le dépassement de capacité ne peut plus simplement se gérer avec l'indicateur de retenue. Considérons par exemple l'addition de deux nombres de n bits a et b, dont le résultat est r, le bit de signe r15 et l'indicateur de retenue C :

a > 0, b > 0, r < 2n-1 
          -->  C = 0, r15 = 0
a > 0, b > 0, r > 2n-1 (Dépassement !) 
          -->  C = 0, r15 = 1
a > 0, b < 0, a > |b| 
Code(a) + Code(b) = a + 2n - |b| = 2n + a-|b|   avec 0 < a-|b| < 2n-1

          -->  C = 1, r15 = 0
a > 0, b < 0, a < |b| 
Code(a) + Code(b) = a + 2n - |b| = 2n - (|b|-a)   avec 0 < |b|-a < 2n-1

          -->  C = 0, r15 = 1
a < 0, b > 0, |a| <\ b 
Code(a) + Code(b) = 2n - |a| + b = 2n + b-|a|   avec 0 < b-|a| < 2n-1

          -->  C = 1, r15 = 0
a < 0, b > 0, |a| > b 
Code(a) + Code(b) = 2n - |a| + b = 2n - (|a|-b)   avec 0 < |a|-b < 2n-1

          -->  C = 0, r15 = 1
a < 0, b < 0, |a|+|b| > 2n-1 (Dépassement !) 
Code(a) + Code(b) = 2n - |a| + 2n - |b| = 2n + 2n - (|a+b|) avec |a+b| > 2n-1

          -->  C = 1, r15 = 0
a < 0, b < 0, |a|+|b| < 2n-1 
Code(a) + Code(b) = 2n - |a| + 2n - |b| = 2n + 2n - (|a+b|) avec 0 < |a+b| < 2n-1

          -->  C = 1, r15 = 1
  Récapitulatif des conditions de dépassement  
  a15 b15 r15    opérandes    résultat    retenue    dépassement  
  0    0    0    a > 0 b > 0    r > 0    0    non  
  0    0    1    a > 0 b > 0    r < 0    0    OUI  
  0    1    0    a > 0 b < 0    r > 0    1    non  
  0    1    1    a > 0 b < 0    r < 0    0    non  
  1    0    0    a < 0 b > 0    r > 0    1    non  
  1    0    1    a < 0 b > 0    r < 0    0    non  
  1    1    0    a < 0 b < 0    r > 0    1    OUI  
  1    1    1    a < 0 b < 0    r < 0    1    non  

Le récapitulatif indique très clairement que l'indicateur de retenue (carry flag) ne peut pas être utilisé comme indicateur de dépassement en arithmétique signée. Il faut utiliser un autre indicateur de dépassement (overflow flag), qui obéit à l'équation : 

     __  __                __
OF = a15.b15.r15 + a15.b15.r15
En additionnant par exemple deux opérandes de 16 bits :

si OF est à 0, le bit de poids fort (i.e. r15) donne le signe du résultat dont la valeur est disponible sur les 15 bits de poids faible.

si OF est à 1, l'indicateur de retenue (C) donne le signe du résultat qui est lui-même sur 16 bits. Dans ce dernier cas, le bit de poids fort ne donne pas le signe du résultat !

 

Conclusion

Le code complément à 2 présente de nombreux avantages : il y a un seul code pour 0, l'addition de deux nombres se fait simplement en additionnant leurs codes, et il est très simple d'obtenir l'opposé d'un nombre... Ceci explique que tous les processeurs actuels utilisent ce code pour effectuer des calculs en arithmétique signée, et disposent de fonctions spéciales pour l'implémenter (cf. indicateur de dépassement).
