Systèmes de numération


Entiers signés

1.3 Code complément à 1

Principe

Ce système est utilisé pour représenter des nombres positifs ou négatifs. En supposant que l'on dispose de n bits, les nombres positifs de 0 à 2n-1-1 sont représentés en code binaire naturel : 

X > 0   -->   code(x) = x

Les nombres négatifs de valeur absolue inférieure à 2n-1 sont représentés par : 

X < 0   -->   code(x) = 2n - 1 - |x|
On peut remarquer que la condition |x| < 2n-1 résulte en un code toujours supérieur au plus grand positif : 
|x| < 2n-1   -->   code(x) = 2n - 1 - |x|  >  2n - 1 - 2n-1 = 2n-1 - 1
Exemple

La droite de 4 bits donne les codes des nombres -7 à +7 : 

     -7                  -1    -0    +0    +1                  +7  
-----+----- - - - - -----+-----+-----+-----+----- - - - - -----+-->
   1000                1110  1111  0000  0001                0111

8 bits couvrent l'intervalle [-127,+127], 16 bits couvrent [-32767,+32767].

 

Propriétés

La somme des codes d'un nombre et de son opposé donne un mot binaire dont tous les bits sont à 1 : 
Code(x) + Code(-x) = x + 2n - 1 - x = 2n - 1
Or, 2n-1 est le plus grand nombre qui puisse être représenté avec n bits, dont tous ses bits sont à 1. Voici un exemple : 
Code(5) + Code(-5) = 0101 + 1010 = 1111

 

Corollaire : De la propriéte précédente il découle que : 
           _______                   ______ 
Code(x) =  Code(-x)   and Code(-x) = Code(x)
Pour obtenir le code d'un nombre négatif, il suffit de complémenter le code de sa valeur absolue.

 

Addition

L'addition de deux nombres positifs ne pose pas de problème car il suffit d'additionner leurs codes : le code du résultat est la somme des codes des opérandes.
 
Soient deux nombres de signe opposé tels que : 
x > 0,  y < 0   /   |x| > |y|
En essayant d'additionner leurs codes, on obtient : 
Code(x) + Code(y) = x + 2n - 1 - |y|                   
                  = 2n - 1 + (x+y)         avec x+y > 0
                  = 2n - 1 + Code(x+y)
Soit encore : 
Code(x+y) = Code(x) + Code(y) + 1 - 2n
Le dernier terme de cette équation ne modifie que le (n+1)ième bit du résultat (en fait codé sur n bits), ce qui permet d'écrire : 
Code(x+y) = Code(x) + Code(y) + 1
Le code de la somme de deux nombres de signe différent est obtenu en incrémentant la somme des codes des deux opérandes. (S).

 

Par souci de rigueur, essayons deux nombres tels que : 
x > 0,  y < 0   /   |x| < |y|
En additionnant leurs codes on obtient : 
Code(x) + Code(y) = x + 2n - 1 - |y|                   
                  = x + 2n - 1 + (x+y)     avec x+y < 0
                  = x + 2n - 1 - |x+y|                 
                  = Code(x+y)
Cette dernière équation montre que la proposition (S) est tout simplement fausse. Dans certains cas le code de la somme est simplement obtenu par addition des codes des opérandes, dans d'autres cas il faut y ajouter 1.

 

N'oublions pas le cas de l'addition de deux négatifs : 
Code(x) + Code(y) = 2n - 1 - |x| + 2n - 1 - |y|                           
                  = 2n - 1 + 2n - 1 + (x+y)                               
                  = 2n - 1 + 2n - 1 - |x+y|       car x+y < 0             
                  = 2n - 1 + Code(x+y)            |x+y| <= 2n-1
On en déduit : 
Code(x+y) = Code(x) + Code(y) + 1 - 2n
Comme le dernier terme de cette équation modifie uniquement le (n+1)ième bit du résultat, nous pouvons à nouveau écrire : 
Code(x+y) = Code(x) + Code(y) + 1

Avantages et inconvénients

Il y a toujours deux codes différents pour le nombre 0.
L'addition est assez compliquée à mettre en oeuvre puisqu'il faut éventuellement incrémenter le code obtenu en fonction du signe du résultat.
La gestion des dépassements n'est pas évidente (cf. les facteurs 2n ci-dessus).
Un avantage de ce code est que la soustraction peut être effectuée par addition de l'opposé, qui est simplement obtenu par complémentation du code du nombre considéré.
