Exemple XML

Entité Générale Externe Analysée

Voici un document XML faisant appel à une entité générale externe : 

<?xml version="1.0" encoding="utf-8"?>

<!DOCTYPE classe [
	<!ELEMENT classe (promo,étudiant+)>
	<!ELEMENT étudiant (prénom,nom,email)>

	<!ENTITY filles SYSTEM "ent-parsed-ext-exemple1-1.ent">
]>

<classe>
 <étudiant>
  <prénom>Raymond</prénom>
  <nom>Deubaze</nom>
  <email>Raymond.Deubaze@ec-lyon.fr</email>
 </étudiant>

 &filles;

</classe>

L'entité externe correspond aux informations suivantes (remarquer qu'il ne s'agit pas d'un document XML en soi, qui aurait sinon deux éléments racine) : 

 <étudiant>
  <prénom>Ginette</prénom>
  <nom>Ringard</nom>
  <email>ginette@wanadoo.fr</email>
 </étudiant>

 <étudiant>
  <prénom>Elsa</prénom>
  <nom>Plique</nom>
  <email>elsaplik@yahoo.com</email>
 </étudiant>

Quelques remarques:

  • la plupart des navigateurs sont très décevants car ils ne vont pas chercher le contenu de l'entité externe,
  • l'entité externe seule n'est pas un document XML en tant que tel, et ne peut donc pas être lue seule par un parseur,
  • ce document, muni de son entité externe, est un document valide :

Test du validateur

Afin de tester le validateur, introduisons une erreur au sein de l'entité externe (mise à part la référence à l'entité externe, le document principal n'est pas modifié) : 

 <étudiante>
  <prénom>Ginette</prénom>
  <nom>Ringard</nom>
  <email>ginette@wanadoo.fr</email>
 </étudiante>

 <étudiante>
  <prénom>Elsa</prénom>
  <nom>Plique</nom>
  <email>elsaplik@yahoo.com</email>
 </étudiante>
