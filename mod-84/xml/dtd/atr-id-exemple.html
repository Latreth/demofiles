Attribut référence d'identifiant unique

La définition ci-dessous déclare un élément login dont l'attribut accesslist doit prendre pour valeur une liste d'identifiants uniques : 

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE rights  [

	<!ELEMENT rights (role+,login+)>

	<!ELEMENT role (#PCDATA)>
	<!ATTLIST role id ID #REQUIRED>

	<!ELEMENT login (#PCDATA)>
	<!ATTLIST login accesslist IDREFS #REQUIRED>
]>

Le document ci-dessous est valide puisqu'il respecte cette DTD : 

<rights>
 <role id="admin">Administrateur Système</role>
 <role id="webmaster">Administrateur Web</role>
 <role id="baseroot">Administrateur MySQL</role>
 <role id="user">Utilisateur sans droits</role>

 <login accesslist="admin webmaster baseroot">root</login>
 <login accesslist="user webmaster baseroot">muller</login>
 <login accesslist="user">rdeubaze</login>
</rights>

Le document ci-dessous, par contre, n'est pas valide par rapport à cette DTD. En effet, la valeur de l'attribut accesslist de l'un des éléments ne fait pas référence à un identifiant unique : 

<rights>
 <role id="admin">Administrateur Système</role>
 <role id="webmaster">Administrateur Web</role>
 <role id="baseroot">Administrateur MySQL</role>
 <role id="user">Utilisateur sans droits</role>

 <login accesslist="admin webmaster baseroot">root</login>
 <login accesslist="user webmaster baseroot">muller</login>
 <login accesslist="user dieu">rdeubaze</login>
</rights>
